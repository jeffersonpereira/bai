# Twin Plan — Agente de WhatsApp Conversacional
**Selected Specialist:** twin-backend
**Selection Reason:** keyword (api + backend + webhook)
**Timestamp:** 2026-05-08T00:00:00
**Quality Level:** pragmatic

---

## Objetivo
Implementar um agente de IA que intercepta mensagens recebidas no webhook do WhatsApp, responde automaticamente ao interessado com contexto do imóvel, qualifica o lead com perguntas naturais e atualiza o status no CRM.

---

## Análise do Estado Atual

### Fluxo atual do webhook (`POST /whatsapp/webhook/message`):
1. Recebe payload do serviço Node.js
2. Deduplica por `message_id`
3. Salva `WhatsAppMessage` com `direcao="entrada"`
4. Se primeira mensagem do contato → cria lead via `criar_lead_auto()`
5. Retorna `{"ok": True}`

### Lacunas:
- Nenhuma resposta automática é enviada
- Lead fica em `situacao="novo"` indefinidamente
- Corretor precisa agir manualmente para cada contato

### Dados disponíveis para o agente:
- Histórico de mensagens: `WhatsAppMessage` filtrado por `jid_conversa`
- Dados do imóvel: modelo `Imovel` (titulo, preco, cidade, bairro, quartos, area, tipo_imovel, descricao)
- Dados do lead: modelo `Lead` (nome, telefone, situacao, observacoes)
- Envio de resposta: `POST /sessions/{user_id}/send` (Node service, já usado em `send_message`)

---

## Arquitetura do Agente

```
Webhook recebe mensagem
        ↓
Salva mensagem + (opcionalmente) cria lead
        ↓
[NOVO] Chama whatsapp_agent.handle_incoming_message() em background
        ↓
Agente busca histórico da conversa (últimas 10 msgs)
        ↓
Agente busca imóvel associado ao lead (se houver)
        ↓
Chama Claude API com system prompt + histórico + contexto
        ↓
Claude decide: responder / atualizar lead / encerrar automação
        ↓
Agente executa ações (envia mensagem via Node, atualiza lead no DB)
        ↓
Retorna (corretor vê lead atualizado no Kanban)
```

### Condições para NÃO responder automaticamente:
- Corretor enviou mensagem manualmente nas últimas 2 horas (evita sobreposição)
- Lead já está em `situacao` != "novo" e != "contatado" (visita/proposta/fechado)
- Agente já respondeu 5+ vezes sem qualificação (evita loop)

## Arquivos a Criar/Modificar

### 1. CRIAR `backend/app/agents/whatsapp_agent.py` (arquivo principal)
- Classe `WhatsAppAgent` com método `handle_incoming_message()`
- Busca histórico de mensagens e contexto do imóvel
- Monta system prompt com persona do corretor
- Chama `anthropic.messages.create()` com histórico como `messages`
- Usa tool_use com 2 ferramentas: `send_whatsapp_reply` e `update_lead_status`
- Executa as ferramentas retornadas pelo modelo

### 2. MODIFICAR `backend/app/core/config.py`
- Adicionar campo `ANTHROPIC_API_KEY: str = ""`

### 3. MODIFICAR `backend/app/routers/whatsapp.py`
- Importar `whatsapp_agent`
- No `webhook_message()`, após salvar a mensagem, chamar o agente de forma assíncrona (via `asyncio.create_task` ou `BackgroundTasks`)

### 4. MODIFICAR `backend/requirements.txt`
- Adicionar `anthropic>=0.40.0`

## Implementação Detalhada

### `whatsapp_agent.py` — Lógica completa:

**System Prompt (template):**
```
Você é um assistente imobiliário da {nome_corretor}. Seu papel é responder 
perguntas sobre imóveis de forma natural, qualificar interessados e agendar 
visitas. Seja cordial, direto e use linguagem informal mas profissional.

{contexto_imovel}

Qualifique o interessado descobrindo:
1. Orçamento disponível
2. Prazo para compra/aluguel
3. Necessidade de financiamento

Quando tiver essas informações, use update_lead_status com "contatado".
Ofereça agendar uma visita quando o lead estiver qualificado.
Não invente informações sobre o imóvel que não estejam no contexto.
```

**Ferramentas (tool_use):**
```python
tools = [
    {
        "name": "send_whatsapp_reply",
        "description": "Envia uma mensagem de resposta para o contato no WhatsApp",
        "input_schema": {
            "type": "object",
            "properties": {
                "message": {"type": "string", "description": "Texto da resposta"}
            },
            "required": ["message"]
        }
    },
    {
        "name": "update_lead_status",
        "description": "Atualiza o status do lead no CRM após qualificação",
        "input_schema": {
            "type": "object",
            "properties": {
                "situacao": {
                    "type": "string",
                    "enum": ["novo", "contatado", "visita", "proposta", "perdido"],
                    "description": "Novo status do lead"
                },
                "observacoes": {"type": "string", "description": "Notas sobre a qualificação"}
            },
            "required": ["situacao"]
        }
    }
]
```

## QA / Validação

- [ ] Agente não responde se corretor enviou mensagem manual recentemente
- [ ] Agente não entra em loop (máx 5 respostas automáticas por conversa)
- [ ] Lead é atualizado de "novo" → "contatado" após qualificação
- [ ] Mensagem de resposta salva no banco como `direcao="saida"`
- [ ] Se `ANTHROPIC_API_KEY` vazia → agente loga warning e não responde (sem quebrar webhook)
- [ ] Imóvel sem associação → agente usa prompt genérico sem contexto de imóvel específico

---

## Rollback
- Reverter `whatsapp.py` para não chamar o agente
- Agente é chamado em background — falha não afeta o webhook principal
