## Visão Geral

O anunciante (corretor) que cadastrou um imóvel deve ser o único capaz de atribuir e desatribuir corretores àquele imóvel, e o único capaz de anexar documentos a ele. Atualmente, qualquer imobiliária pode manipular atribuições em imóveis de terceiros e qualquer usuário autenticado pode pendurar documentos em qualquer imóvel pelo ID. Os dois gaps críticos ficam no serviço de imóveis; o gap moderado fica no serviço de documentos; o gap baixo é uma exposição pública de dados de agendamento.

### GAP 1 — assign_broker sem verificação de ownership

**Arquivo:** `backend/app/services/imovel_service.py`
**Função:** `assign_broker`
**Linha:** 498–506

**Problema:** `get_property()` retorna o imóvel mas nada verifica se `current_user` (a imobiliária autenticada) é dona daquele imóvel. Qualquer imobiliária consegue atribuir corretores em imóveis cadastrados por outras imobiliárias.

```python
# ANTES (linhas 498-506):
def assign_broker(db: Session, property_id: int, user_id: int, current_user: Usuario) -> dict:
    get_property(db, property_id)
    broker = db.query(Usuario).filter(Usuario.id == user_id, Usuario.imobiliaria_id == current_user.id).first()
    if not broker:
        raise HTTPException(status_code=404, detail="Corretor não encontrado na equipe")
    assignment = ResponsavelImovel(imovel_id=property_id, usuario_id=user_id)
    db.merge(assignment)
    db.commit()
    return {"message": f"Imóvel atribuído a {broker.nome}"}

# DEPOIS:
def assign_broker(db: Session, property_id: int, user_id: int, current_user: Usuario) -> dict:
    prop = get_property(db, property_id)
    # Verifica que a imobiliária autenticada é dona do imóvel:
    # O imóvel pertence diretamente à imobiliária OU a um corretor desta imobiliária
    owns = (
        prop.corretor_id == current_user.id
        or (prop.corretor is not None and prop.corretor.imobiliaria_id == current_user.id)
    )
    if not owns:
        raise HTTPException(status_code=403, detail="Sem permissão: imóvel pertence a outra imobiliária")
    broker = db.query(Usuario).filter(Usuario.id == user_id, Usuario.imobiliaria_id == current_user.id).first()
    if not broker:
        raise HTTPException(status_code=404, detail="Corretor não encontrado na equipe")
    assignment = ResponsavelImovel(imovel_id=property_id, usuario_id=user_id)
    db.merge(assignment)
    db.commit()
    return {"message": f"Imóvel atribuído a {broker.nome}"}
```

### GAP 3 — create_document sem verificação de ownership do imóvel

**Arquivo:** `backend/app/services/documento_service.py`
**Função:** `create_document`
**Linha:** 49–70

**Problema:** Qualquer usuário autenticado pode criar um documento com `imovel_id` de qualquer imóvel. O campo `imovel_id` vem direto do schema `DocumentoCriar` (linha 20 do schema) sem validação de posse.

A função precisa importar `Imovel` e verificar ownership quando `imovel_id` está presente.

```python
# ANTES — corpo de create_document (linhas 49-70):
def create_document(db: Session, doc_in: DocumentoCriar, current_user: Usuario) -> Documento:
    data = doc_in.model_dump()
    _validate_fields(data)

    # Deduplicação por hash
    if data.get("hash_sha256"):
        existing = db.query(Documento).filter(
            Documento.hash_sha256 == data["hash_sha256"],
            Documento.enviado_por == current_user.id,
        ).first()
        if existing:
            raise HTTPException(
                status_code=409,
                detail=f"Arquivo duplicado: documento '{existing.titulo}' já possui este conteúdo",
            )

    doc = Documento(**data, enviado_por=current_user.id)
    db.add(doc)
    db.commit()
    db.refresh(doc)
    doc.total_versoes = 1
    return doc

# DEPOIS — adicionar bloco de verificação de ownership após _validate_fields:
def create_document(db: Session, doc_in: DocumentoCriar, current_user: Usuario) -> Documento:
    data = doc_in.model_dump()
    _validate_fields(data)

    # Verificação de ownership do imóvel quando imovel_id é fornecido
    if data.get("imovel_id") and current_user.perfil not in ("admin", "imobiliaria"):
        prop = db.query(Imovel).filter(Imovel.id == data["imovel_id"]).first()
        if not prop:
            raise HTTPException(status_code=404, detail="Imóvel não encontrado")
        if prop.corretor_id != current_user.id:
            raise HTTPException(status_code=403, detail="Sem permissão: você não é o anunciante deste imóvel")

    # Deduplicação por hash
    if data.get("hash_sha256"):
        existing = db.query(Documento).filter(
            Documento.hash_sha256 == data["hash_sha256"],
            Documento.enviado_por == current_user.id,
        ).first()
        if existing:
            raise HTTPException(
                status_code=409,
                detail=f"Arquivo duplicado: documento '{existing.titulo}' já possui este conteúdo",
            )

    doc = Documento(**data, enviado_por=current_user.id)
    db.add(doc)
    db.commit()
    db.refresh(doc)
    doc.total_versoes = 1
    return doc
```

## QA Validation Steps

1. **GAP 1 — assign:** Autenticar como imobiliária A, tentar `POST /imoveis/{id_imovel_de_B}/assign?user_id=X` — deve retornar `403 Sem permissão: imóvel pertence a outra imobiliária`. Com imobiliária dona do imóvel, a mesma chamada deve retornar `200`.

2. **GAP 2 — unassign:** Autenticar como imobiliária A, tentar `DELETE /imoveis/{id_imovel_de_B}/assign/{user_id}` — deve retornar `403`. Com a imobiliária correta, deve retornar `200` e remover a atribuição.

3. **GAP 2 — assinatura:** Confirmar que `unassign_broker` no serviço aceita o 4º argumento `current_user` sem `TypeError`. Rodar `pytest` ou checar manualmente a importação.

4. **GAP 3 — documento ownership — corretor sem permissão:** Autenticar como corretor B (não dono do imóvel), tentar `POST /documentos/` com `imovel_id` de imóvel do corretor A — deve retornar `403 Sem permissão: você não é o anunciante deste imóvel`.

5. **GAP 3 — documento ownership — corretor dono:** Autenticar como corretor A (dono do imóvel), `POST /documentos/` com `imovel_id` correto — deve retornar `201`.

6. **GAP 3 — documento ownership — admin/imobiliaria bypass:** Autenticar como admin ou imobiliária, `POST /documentos/` com qualquer `imovel_id` — deve retornar `201` (sem checagem de ownership para estes perfis).

7. **GAP 4 — availability sem token:** Fazer `GET /imoveis/{id}/availability` sem `Authorization` header — deve retornar `401 Not authenticated`.

8. **GAP 4 — availability com token:** Mesma chamada com token válido de qualquer perfil — deve retornar `200` com a lista de janelas.
