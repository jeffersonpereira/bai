## Bloqueadores Críticos (DEVEM ser resolvidos antes de produção)

### SEG-01 — Credenciais reais no `.env` versionado
- **Arquivo:** `backend/.env`
- **Risco:** Acesso não autorizado ao banco e Supabase
- **Ação:** Revogar credenciais expostas + adicionar `.env` ao `.gitignore`

### SEG-02 — SECRET_KEY fraca
- **Valor atual:** `bai_super_secret_key_2024_producao_troque_agora_32x`
- **Risco:** Tokens JWT forjáveis por atacantes
- **Ação:** `python -c "import secrets; print(secrets.token_hex(32))"`

### SEG-03 — Supabase Service Key exposta
- **Risco:** Acesso root ao banco de dados
- **Ação:** Revogar e regenerar no painel Supabase

### SEG-04 — `DEBUG=true` no `.env`
- **Risco:** Swagger UI e stack traces expostos publicamente
- **Ação:** `DEBUG=false` em produção

## Plano de Correção Prioritizado

### Sprint 1 — Bloqueadores de Segurança (1-2 dias)
1. Revogar todas as credenciais do `.env` (DB, Supabase)
2. Adicionar `.env` ao `.gitignore` e remover do histórico git
3. Gerar nova `SECRET_KEY` forte
4. Definir `DEBUG=false` para produção
5. Criar `.env.production.example` com todas as vars necessárias

### Sprint 2 — Estabilidade (3-5 dias)
1. Fixar versões no `requirements.txt` com `pip freeze`
2. Adicionar SDKs de pagamento ao `requirements.txt` (stripe, mercadopago)
3. Corrigir testes unitários (`test_map_bbox.py` → importar `imovel_service`)
4. Corrigir ou desativar ScoutAgent (importação de modelos legados)
5. Alinhar quota do plano gratuito (código vs. marketing): definir 3 ou 5 imóveis

### Sprint 3 — Observabilidade e Infra (1 semana)
1. Implementar logging estruturado com `loguru` no backend
2. Criar `Dockerfile` para backend Python
3. Criar `Dockerfile` para frontend Next.js
4. Criar `docker-compose.yml` para ambiente de desenvolvimento
5. Configurar Sentry para monitoramento de erros

### Sprint 4 — Auth e Segurança Adicional (3-5 dias)
1. Implementar endpoint `/auth/refresh` para renovação silenciosa de token
2. Migrar JWT de `localStorage` para `httpOnly cookie`
3. Adicionar validação mínima de senha no backend (8+ chars)
4. Implementar CSP (Content Security Policy) no Next.js

### Sprint 5 — CI/CD e Testes (1-2 semanas)
1. Criar GitHub Actions: lint + testes + deploy
2. Escrever testes de integração para endpoints críticos (auth, checkout, CRM)
3. Configurar ruff para linting Python
4. Meta: cobertura de testes > 30%

### Fase futura — Requisitos de Mercado
1. Integração com portais imobiliários (ZAP, Viva Real)
2. Assinatura digital de contratos
3. Portal do proprietário
4. Serviço de email (Resend/SendGrid)
5. Notificações push
