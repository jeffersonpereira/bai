# Twin Plan — Fix: Input de valor sobrepondo prefixo R$ no modal de proposta

```
Selected Specialist: frontend-react
Selection Reason: heuristic (UI/layout/CSS bug)
Timestamp: 2026-05-06
Quality Level: pragmatic
```

## Visão Geral

O modal de proposta exibe um input de valor com prefixo "R$" posicionado via `absolute`. O input usa a classe `.input-base` + `pl-10` (Tailwind). Porém, como `globals.css` usa `@import "tailwindcss"` na linha 1 e define `.input-base` depois (linha 200), o shorthand `padding: 0.625rem 1rem` de `.input-base` sobrescreve o `pl-10` do Tailwind na cascata CSS. Resultado: padding-left efetivo = 1rem = 16px, idêntico ao `left-4` do prefixo, causando sobreposição do "R$" com o texto digitado.

---

## FASE 1 — CORREÇÃO PRINCIPAL

### FIX 1 — Mover `.input-base` para `@layer components` em globals.css

**Arquivo:** `frontend/src/app/globals.css`  
**Linhas:** 199–217

Ao envolver `.input-base` (e seus seletores derivados `::placeholder` e `:focus`) em `@layer components {}`, o Tailwind garantirá que utilities (`pl-10`, etc.) sempre tenham precedência sobre estilos de componente, conforme a arquitetura de layers do Tailwind v4.

```css
/* ANTES (linhas 199-217): */
/* ─── Input base ────────────────────────────────────────────────── */
.input-base {
  width: 100%;
  padding: 0.625rem 1rem;
  background: #ffffff;
  border: 1px solid var(--color-border);
  border-radius: var(--radius-xl);
  font-size: clamp(0.875rem, 1vw + 0.5rem, 1rem);
  color: var(--color-text-primary);
  transition: border-color var(--transition-fast), box-shadow var(--transition-fast);
  outline: none;
}
.input-base::placeholder {
  color: var(--color-text-muted);
}
.input-base:focus {
  border-color: var(--color-primary);
  box-shadow: 0 0 0 3px rgba(37,99,235,.12);
}

/* DEPOIS: */
/* ─── Input base ────────────────────────────────────────────────── */
@layer components {
  .input-base {
    width: 100%;
    padding: 0.625rem 1rem;
    background: #ffffff;
    border: 1px solid var(--color-border);
    border-radius: var(--radius-xl);
    font-size: clamp(0.875rem, 1vw + 0.5rem, 1rem);
    color: var(--color-text-primary);
    transition: border-color var(--transition-fast), box-shadow var(--transition-fast);
    outline: none;
  }
  .input-base::placeholder {
    color: var(--color-text-muted);
  }
  .input-base:focus {
    border-color: var(--color-primary);
    box-shadow: 0 0 0 3px rgba(37,99,235,.12);
  }
}
```

## FASE 2 — MELHORIA COMPLEMENTAR

### FIX 2 — Ocultar spinners nativos do `type="number"` no input de valor

**Arquivo:** `frontend/src/app/properties/[id]/page.tsx`  
**Linha:** 261

O `type="number"` exibe setas nativas do browser que podem conflitar visualmente com o prefixo "R$". Adicionar classes Tailwind para ocultar os spinners.

```tsx
/* ANTES (linha 261): */
className="input-base pl-10"

/* DEPOIS: */
className="input-base pl-10 [appearance:textfield] [&::-webkit-outer-spin-button]:appearance-none [&::-webkit-inner-spin-button]:appearance-none"
```
